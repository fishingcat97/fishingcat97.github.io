## 命名规范
- 类、枚举
  - 名词或名词短语（不应使用动词）
  - 使用大驼峰
- 接口
  - 名词或名词短语
  - 形容词或形容词短语
- 测试类
  - 正在测试的类名称开头，以Test结尾
- 抽象类
  - 以Abstract或Base开头
- 方法名
  - 小驼峰命名
  - 动词或动词短语
    - get + 非布尔属性名()
    - is/has/can/should... + 布尔属性名()
    - set + 属性名()
    - has + 名词/形容词()
    - 动词
    - 动词 + 宾语()
    - 回调方法(callback) 允许介词 + 动词形式命名，如：onCreate，onDestory，toString
- 测试方法名
  - 可使用下划线，建议BDD
- 常量
  - 由全大写单词组成，单词用下划线分离
  - 不要使用魔鬼数字（不应该取NUM_FIVE = 5）
- 非常量字段
  - 小驼峰
  - 名词或名词短语
  - 集合用复数
